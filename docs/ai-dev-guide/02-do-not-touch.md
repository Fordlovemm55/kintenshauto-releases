# 02 — Do Not Touch

Read this BEFORE editing any file. Each rule below comes from an actual production breakage or build failure.

## Generated / downloaded folders — your edits will be lost

| Path | Generated by | If you edit |
|---|---|---|
| `dist/` | `npm run build-frontend` (Vite) | Overwritten on next build |
| `dist/assets/index-*.js` | Vite | Overwritten — edit `src/` instead, then rebuild |
| `dist-installer/` | `npx electron-builder` | Overwritten on next installer build |
| `dist-installer/win-unpacked/` | electron-builder | Overwritten on next installer build |
| `node_modules/` | `npm install` | Wiped on next `npm install` — use `patch-package` if you need to modify |
| `node_modules/better-sqlite3/build/` | electron-builder's `install-app-deps` | Rebuilt on next install |
| `bin/win32/` | `scripts/download-deps.js` (auto on first launch) | Replaced on next user-install; add your custom binary to `scripts/` instead |
| `dist-installer/v1.0.0/` | Manual copy from earlier builds | Disposable; recreate as needed |

## Behavior — DO NOT change these without a strong reason

### `src/backend/core/poster.js` — `COMPOSER_URL` constant
The Facebook composer URL. Currently tested-working. Changing it usually breaks posting silently — the page loads but the submit handler can't find the right form elements.

### `src/backend/core/` (entire folder)
The FB automation primitives (poster, orchestrator, worker, scout, browserManager, peakSchedule).
- **Structural refactors** (rename functions, extract files) — OK if you re-test the post pipeline end-to-end.
- **Behavioral changes** (timing, retry counts, selector waits, click sequences) — NOT OK without an explicit user ask + test plan.

### `pending_approvals` table — UNIQUE composite `(watched_id, video_id)`
The scout uses this constraint to dedupe found clips. If you change it (e.g. add `created_at` to the index), duplicate clips will start appearing in the review queue.

### SQLite migrations — additive only
SQLite can't `ALTER COLUMN` to change a type. If you must change a column, do it via:
1. Create new table with the new schema
2. `INSERT INTO new SELECT ... FROM old`
3. `DROP old; RENAME new TO old`

Do NOT try to write `ALTER TABLE ... ALTER COLUMN ...` — it will silently fail in some SQLite versions and crash on others.

### `better-sqlite3` dependency version — keep at `^12.x`
Node 24 has no prebuilts for `better-sqlite3` v11. Downgrading will force every developer to compile from source (requires Visual Studio Build Tools on Windows). Stay on v12+.

### Encryption key file at `%APPDATA%/KINTENSHAUTO/.encryption-key`
- Per-install random key, used for FB credentials AND the session store
- If deleted, the user loses their saved FB profile credentials AND has to re-login
- Migration code that touches this file MUST preserve the existing key

## Git / Distribution

### `Fordlovemm55/kintenshauto-releases` (public GitHub repo)
- Do NOT delete or edit assets on an existing release that users have already downloaded — electron-updater verifies sha512 in `latest.yml`. Mismatch = "update failed" for all users on that version.
- Do NOT make the repo private without first removing the embedded PAT from `electron/main.js` and rebuilding — public repo means PAT is no longer needed, but if you flip to private without the rebuild, all existing installs fail to fetch updates.

### Supabase `app_versions` table
- `min_required = true` means force update. Setting it on the WRONG version (e.g. a version that hasn't been built + uploaded to GitHub yet) will lock every user in a broken update loop ("update available" → tries to fetch → 404 from GitHub).
- Always upload the GitHub release FIRST, verify it returns 200 on the download URL, THEN insert the `app_versions` row.

### Publishing a release without user approval
The user has stated explicitly: do not publish a new version (GitHub release + `app_versions` insert) until they say "อัพ" / "publish" / "push release". See [04-release-workflow.md](04-release-workflow.md).

## Sensitive files — never commit

| File | What it contains |
|---|---|
| `.env` | `KINTENSHAUTO_SUPABASE_URL`, `KINTENSHAUTO_SUPABASE_ANON_KEY` — already gitignored |
| `.session` (in user data) | Encrypted Supabase access token — never read on disk; use `authService.getStoredSession()` |
| `.encryption-key` (in user data) | AES-256 key for the .session + FB password encryption |
| `kintenshauto-cloud/PROJECT.md` | Supabase service_role + DB password — gitignored |
| `kintenshauto-admin/.env.local` | `SUPABASE_SERVICE_ROLE_KEY`, `ADMIN_SHARED_SECRET` — gitignored |

If a token or service key ever appears in chat or a commit, treat it as compromised and rotate immediately.

## Destructive shell commands — never run these without explicit confirmation

```bash
# Wipes files that aren't tracked in git
git clean -fdx

# Deletes branches without warning
git push --force
git push --force-with-lease   # safer but still destructive

# Mirror-syncs (deletes files in dest not present in source)
robocopy <src> <dest> /MIR    # Use /E (recursive, no delete) instead

# Drops a Supabase table — IRREVERSIBLE in production
DROP TABLE ...

# Wipes node_modules
rm -rf node_modules           # Slow to recover; only do if package-lock conflicts demand it

# Skips git hooks (bypasses our local test runner + commit-msg formatter)
git commit --no-verify
```

## Things that look safe but bite

| Action | Why it bites |
|---|---|
| Running `npm install` without `SKIP_POSTINSTALL=1 --ignore-scripts` | Triggers ffmpeg/yt-dlp download (~3-5min) |
| `npx electron-builder --win --dir` then renaming the .exe | electron-updater's `latest.yml` references the original filename; rename breaks update detection |
| Editing a file in `electron/` without restarting the app | Electron main process changes need full app restart; HMR doesn't cover them |
| Changing `electron/preload.js` API surface | All `window.kintenshauto.*` calls in React must be checked — silent runtime errors otherwise |
| Adding a `process.env.X` check to `cloud/config.js` | Tests rely on the test-mode guard; new env reads need a `VITEST !== 'true'` carve-out |
| Importing from `node_modules/@base-ui/react` in admin panel | We use base-nova shadcn variant; some Radix-only props (`asChild`) don't exist — use `render={...}` |
